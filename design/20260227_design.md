This program is a CLI program in Typescript. It is an interactive program. I input the FEN and the side and then it queries Lichess and chess.com to collect the statictics for the next moves played when the user was playing a specific side. It also queries lichess for the opening database.
Lichess and chess.com could be queries at the same time, but two lichess requests should be sequential.
As soon as I input the fen and the side print the board using unicode text characters. The board should have letters and numbers on the side and do not forget that the spaces that are not occupied should be the same width as the board pieces. If it is empty then it is better to just put a dot there, but with the right character width that the board is square.
After showing the board, start doing requests. Write down the status of those three requests.
When finished write down statistics in a text table with the columns:
1. Move in SAN notation (the most common in the total of user games at the top, if no user games exist then most common by Lichess database)
2. The evaluation from Lichess in the python format f"{value:5.2f}"
3. All the other columns have the format <number of games> <White win>/<draw>/<Black win> in percentages with an equivalent python format: f"{value:4.1f}". <number of games> should be aligned right and have the maximum size based on the maximum number of games in the column.
4. The first column is for user games on Lichess, the second column is for user games on chess.com, the third column is for Lichess database games

## Further interactions
After outputting the table allow to input moves in SAN notation. Also allow to press arrow left and enter without anything else inputted to move backwards in the history. After the move was inputted or the arrow pressed then make new requests as described above and output a new table.

Cache the calls that were made during the session and use them instead of calls when possible. Only cache in memory.

The user is named in .env file. If not then ask for the user.
LICHESS_USER=xxx
CHESSCOM_USER=xxx

Cover the code with unit tests when possible.
In the end of the task all the tests should be successful and you should output the unicode board to see how it looks like.
